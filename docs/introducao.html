Introdução

Todas as requisições que utilizamos seguem o formato JSON, tanto para envio, quanto para resposta. E precisam de algumas configurações padrões para que seja possível a comunicação. Segue abaixo mais detalhes.

URL (Caminho Absoluto)

https://paguelogo.com.br/api

Primeiros passos

Para começar a utilizar os nossos serviços REST, deve-se primeiramente fazer a autenticação no sistema. E logo após já pode acessar qualquer funcionalidade a partir de sua URL.

1) Autenticação
2) Acesso às funcinalidades passando os dados recebidos da autenticação nos cabeçalhos das requisições.

Autenticação

Para a autenticação nós usamos o padrão oAuth2. E para tal, a maior parte de nossos endpoints do webservice REST, precisam que mais 2 cabeçalhos de requisição sejam inseridos:

Cabeçalhos

Nome Valor Observação
Authorization Bearer [token] Token de autenticação fornecido através do endpoint "Autenticação"
Whois [código único do usuário] Cada usuário tem um código único que é fornecido através do endpoint de "Autenticação"

Quando a funcionalidade exigir esses cabeçalho, a própria API informará essa necessidade e aqui também na documentação, para cada request, estamos reforçando quando será necessário ou não.

O cabeçalho "Authorization" deve ter o valor no seguinte padrão: "Bearer " + "[token]". Ou seja, deve ter o texto "Bearer" seguido de um espaço em branco, e logo após o token fornecido na autenticação.

O cabeçalho "Whois" é um valor numérico fornecido também na autenticação do usuário.

Funcionalidades abertas

Algumas funcionalidades não precisam de autenticaçao, pois se referem justamente aos meios necessários para se autenticar, ou mesmo para recuperar acesso. Segue abaixo funcionalidades que não necessitam de autenticação.

  • Autenticação
  • Recuperar Senha
  • Contato (Empresa)

Fora essas funcionalidades acima, todas as outras necessitam que o usuário esteja autenticado.
